# 自己管理アプリ 要件定義

## 1. 目的・背景
- 個人がスマートフォン上で予定・タスク・振り返り・翌日の目標設定を一元管理できる自己管理アプリを作成する。
- ストア公開は予定せず、開発者本人が自分用として利用する。

## 2. 想定ユーザー
- 開発者本人（個人利用）。
- スマートフォン（iOS/Android）での利用を想定。

## 3. プラットフォーム・技術
- フロントエンド：Next.js または純粋な React をベースにモバイルラッパー（例：React Native, Expo など）を組み合わせてスマホ向け UI を構築。
- バックエンド／データストア：Supabase。認証・DB・ストレージ・リアルタイム同期などを活用。
- Google カレンダー API と双方向連携。

## 4. 機能要件
### 4.1 予定管理
- Google カレンダーと双方向同期（閲覧・作成・更新・削除）。
- カレンダー表示（リスト/日/週などスマホに最適化したレイアウト）。
- 予定詳細（タイトル、日時、場所、メモなど）の表示・編集。

### 4.2 タスク管理
- 日単位・週単位などのタスク一覧。
- タスクのステータス（未着手/進行中/完了）管理。
- 優先度やカテゴリの付与（任意）。

### 4.3 1日の振り返り
- その日の出来事・感想・達成状況などをテキストで記録。
- 決まった設問テンプレ（例：良かったこと / 改善点 など）をチェックリスト形式で用意。
- テンプレはユーザーがカスタマイズ（項目追加・削除・順序変更）が可能。

### 4.4 翌日の目標設定
- 翌日にフォーカスする目標やタスクを登録。
- 登録時にチェックリストテンプレを活用できる。

### 4.5 通知・リマインダー
- 予定・タスク・振り返り・目標設定のリマインダーをスマホ通知で受け取る。
- 通知タイミングや頻度をユーザーが柔軟に設定可能（例：予定の○分前、毎晩21時に振り返り通知など）。

### 4.6 Google カレンダー連携
- OAuth で認証し、ユーザーのカレンダーとリンク。
- アプリ内で追加・更新した予定は即時に Google カレンダーへ同期。
- Google カレンダー上での変更もアプリへ反映（双方向同期）。

### 4.7 自動保存・クラウド同期
- 入力内容は自動保存され、Supabase 上に同期。
- オフライン時はローカルにキャッシュし、再接続時に同期（将来的な考慮）。

## 5. 非機能要件
- **パフォーマンス**：スマホ利用において快適に動作するレスポンスを目指す。
- **信頼性**：Supabase・Google API 連携の失敗時にユーザーへリトライ手段や状態表示を提供。
- **セキュリティ**：OAuth・Supabase 認証を適切に実装し、個人データを安全に扱う。
- **拡張性**：チェックリスト項目や通知条件などを後から柔軟に追加できる設計。

## 6. 画面イメージ（初期案）
- **ホーム**：今日の予定・タスク・振り返りステータス・翌日目標サマリを1画面で表示。
- **カレンダー**：日/週表示と予定詳細。
- **タスク/目標**：タスク一覧・詳細、翌日目標登録画面。
- **振り返り**：テンプレートを使ったチェックリスト＋自由記述フォーム。

## 7. 今後の検討事項
- Next.js と React Native/Expo の組み合わせ、または別構成の具体的な技術選定。
- 通知実装方式（ネイティブ通知、Push、Supabase Functions 連携など）。
- オフライン状態での扱い（キャッシュ戦略、同期失敗時のUI）。
- UI デザインとアクセシビリティ要件の詳細。

上記は初期ドラフトのため、運用しながら必要に応じて更新する。
