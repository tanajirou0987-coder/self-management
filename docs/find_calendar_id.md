# GoogleカレンダーIDの確認方法

## 📍 確認手順

### 方法1: Googleカレンダーの設定画面から（推奨）

1. **Googleカレンダーを開く**
   - https://calendar.google.com/ にアクセス
   - Googleアカウントでログイン

2. **カレンダーの設定を開く**
   - 左サイドバーで連携したいカレンダーを見つける
   - カレンダー名の横にある **「︙（三点メニュー）」** をクリック
   - **「設定と共有」** を選択

3. **カレンダーIDを確認**
   - 設定画面を下にスクロール
   - **「カレンダーの統合」** セクションを探す
   - **「カレンダーID」** が表示されています
   - カレンダーIDをクリックするとコピーできます
   - または、表示されているテキストを選択してコピー

### 方法2: カレンダーのURLから確認

1. **Googleカレンダーを開く**
2. **カレンダーをクリックして選択**
3. **ブラウザのアドレスバーを確認**
   - URLに `cid=` というパラメータがある場合、その値がカレンダーIDです
   - 例: `https://calendar.google.com/calendar/u/0/r?cid=abc123def456@group.calendar.google.com`

### 方法3: カレンダーの共有設定から確認

1. **カレンダーの設定と共有を開く**
2. **「カレンダーを公開する」セクションを確認**
   - カレンダーを公開している場合、公開URLにカレンダーIDが含まれています
   - 例: `https://calendar.google.com/calendar/ical/カレンダーID/public/basic.ics`

## 📝 カレンダーIDの形式

カレンダーIDは以下のいずれかの形式です：

### プライマリカレンダー（個人のメインカレンダー）
```
あなたのメールアドレス@gmail.com
```
例: `tanaka.daijiro@gmail.com`

### 共有カレンダー
```
ランダムな文字列@group.calendar.google.com
```
例: `abc123def456@group.calendar.google.com`

### その他のカレンダー
- `カスタム名@group.calendar.google.com`
- メールアドレス形式

## 🔍 確認できない場合

### カレンダーの統合セクションが見つからない場合

1. **カレンダーの設定画面を開く**
2. **「一般」タブを確認**
3. **「カレンダーの統合」セクションまでスクロール**
4. まだ見つからない場合は、カレンダーが共有カレンダーではない可能性があります

### プライマリカレンダーの場合

プライマリカレンダー（個人のメインカレンダー）の場合は、あなたのGoogleアカウントのメールアドレスがカレンダーIDです。

例: `your-email@gmail.com`

## ⚠️ 注意事項

- **カレンダーIDは大文字小文字を区別します**
- コピーする際は、余分なスペースが入らないように注意してください
- 共有カレンダーの場合、サービスアカウントに共有設定が必要です

## 📋 次のステップ

カレンダーIDを確認したら：

1. Vercel CLIで環境変数を追加：
   ```bash
   cd app
   vercel env add GOOGLE_CALENDAR_ID production
   ```

2. プロンプトでカレンダーIDを貼り付けてEnter

3. 環境を選択：`Production` → Enter
